### HTMLにつけるclassの命名規則の考え方
参照:[【初学者必見】HTMLにつけるclassの命名規則の考え方解説！](https://pengi-n.co.jp/blog/css-class/)
####  class命名規則のポイント
- class名を見たらどこの何かがわかる
- ある程度の規則性がある

#### その他の注意点

- 英語で付ける

class名は半角英数字で英語で付けるようにしましょう。日本語をローマ字に置き換えるのはNGです。

◯center　☓chuo

- 数字から始まるのはダメ

class名は数字から始まってはいけません。必ずアルファベットで始めるようにしましょう。

◯text01　☓01text

- 複数の単語を組み合わせる場合はハイフンかアンダースコア

複数の単語を組み合わせるときはハイフンまたはアンダースコアでつなぎます。他の単語でつないだり、半角スペースを空けるのはNGです。特に半角スペースを空けた場合、2つの別々のclassとして認識されます。

◯section-title、section\_title　☓section title、section.title

### まずはざっくり場所で名前を付ける

まずはざっくり場所で名前を分けます。例えばheaderにはclass=”header”と付けて、headerの中にある要素にはclass=”header-◯◯”といった感じにclassを付与していくと良いです。

### パーツ名で名前を付ける

パーツ名もclass名の手がかりになります。例えば、ボタンは”◯◯-btn”、写真は”◯◯-img”、本文は”◯◯-text”と付けると良いです。

### 機能で名前を付ける

パーツは機能を持っていることがあるので、機能でclass名を分けても良いです。例えば、送信ボタンなら”submit-btn”、リンクボタンなら”link-btn”といった感じです。

### HTML要素名を利用する

HTML要素の名前を利用することもあります。例えばulは「unordered list」、liは「list item」なので、ulに対して”◯◯-list”、liに対して”◯◯-list-item”とclassを付けると管理しやすくなります。

### 外側を囲むものがほしいとき

コーディングをしていると「この要素達の外側を何かで囲みたい！」と思うことがありますよね。そのときによく使用しているのが”◯◯-container”、”◯◯-wrapper”、”◯◯-outer”です。

### 外側を囲むものの一つ内側がほしいとき

外側をwrapperやouterで囲んだときに「内側にもう一つ囲いがほしい！」と思うこともあるはず。そのときは”◯◯-content”、”◯◯-inner”などとしておけば大丈夫です。

#### ボタン部分を作成 `<a>`
ボタンを作っていきます。ボタンは`<a>`タグで指定します。
しかし`<a>`タグはインライン要素であり、インライン要素には`width`や`height`が指定できないなど不便な点があります。

`<a>`タグは初期状態でインライン要素になっていますが、`display`プロパティを使うと、インラインブロック要素に変更することができます。  
`display`プロパティは`block`（ブロック要素）, `inline-block`（インラインブロック要素）, `inline`（インライン要素）を指定することができます。

#### CSSの記述順序について
CSSの記述順序は、スタイルの適用に大きな影響を与えます。これは「カスケーディング」の一部であり、CSS (Cascading Style Sheets) の名前の由来でもあります。

主に2つの重要な概念があります：「特異度（specificity）」と「ソース順序（source order）」です。

1.  特異度（Specificity）: CSSでは、IDセレクタ、クラスセレクタ、タグセレクタなど、異なる種類のセレクタにはそれぞれ異なる「特異度」があります。IDセレクタは最も特異度が高く、次にクラスセレクタ、そしてタグセレクタと続きます。特異度が高いセレクタのスタイルが、特異度が低いセレクタのスタイルを上書きします。
    
2.  ソース順序（Source Order）: CSSのルールが同じ特異度を持つ場合、その中で後に記述されたルールが優先されます。これは「ソース順序」の原則です。
    

例えば、以下のようなCSSがあるとしましょう：

```css
.button {
  background-color: blue;
}

.button {
  background-color: red;
}

```

この場合、`.button`というクラスを持つ要素の背景色は「赤」になります。なぜなら、両方のルールが同じ特異度（クラスセレクタ）を持つため、ソース順序の原則により後に記述されたルールが優先されるからです。

したがって、CSSの記述順序を変更すると、ページ上の要素のスタイルが変わる可能性があります。これは、特異度が同じセレクタに対するルールがある場合に特に注意が必要です。

#### ヘッダーを上端(0ピクセル)の位置に配置する
**例:**
```css
position: fixed;
top: 0;
```

```css
z-index: 10;
```
このプロパティは、要素のスタッキングコンテキスト内での重ね順（z軸上での位置）を制御します。値が大きいほど、その要素は他の要素よりも前面に表示されます。

### プロパティ「display」
CSSで要素の表示形式を指定する時に使用します。  
要素を横に並べたり、高さや幅を指定したりしたい時に`display`に適切な値を指定していないと思い通りの配置にならないことがあります。

#### 設定できる値
使用頻度の高いのは、`inline`, `block`, `inline-block` ※他多数あり

#### インライン要素
文章の一部として扱われます。  
ex. `<span>`タグ、`<img>`タグなど

ノートの罫線からはみ出さないようなイメージなので、縦幅は指定できないし横幅も中に含んでいる文字数などによって決まります。

#### ブロックレベル要素
文章を覆うひとつのかたまりとして扱われます。  
ex. `<h1>`などの見出しタグ、`<p>`タグ、`<div>`タグなど

横幅が中に文字数などでは決まらず、端から端まで横いっぱいに広がった状態になる要素です。並べると縦に並びます。縦幅を自由に指定できます。

#### インライン要素とブロックレベル要素の関係性
ブロックレベル要素の中にインライン要素を含む決まりになっている。  
逆にインライン要素の中にブロックレベル要素は配置できません。

`display`は、これらのHTMLタグに設定された要素の初期値を変更することができる。

#### では、インラインブロック要素とは
インラインとブロックのいいとこ取りをした状態です。下記のデメリットを解決できます。
```
# デメリット
・インラインでは、`padding`や`margin`を使用すると左右にしか効果がない。  
・ブロックは、横並びにできない。`width`を小さくしても横並びしない。  
```

インラインブロックでは、配置の動き方は「display: inline;」と同様、横に並んでいって、幅いっぱいになった後は行を折り返していきます。  
文字と同じような扱いができるので、例えば文字を真ん中寄せにしたい時に使う「text-align: center」というプロパティ指定が親要素にされていれば、要素は真ん中寄せの配置になります。

#### 使い分けについて
##### インライン
いくつかのタグで初期値として設定されていますが、不自由なのでわざわざインラインを指定するような場面は少ないです。

##### ブロック
全体を覆うブロックとしてまとまりをもたせて、細かい要素を覆う存在として便利です。ここからここまでの領域からは絶対にはみ出してほしくないという時に活用すると、すっきりきれいなレイアウトが作れます。

##### インラインブロック
要素の細かい配置やデザイン調整を行う時に指定します。  
例えば、画面を分割してタイル状に並べたり、背景色を付けてボタンのような要素を作って横に並べたりするような時です。
