・	IAMユーザーを作成しよう

		IAMとは
		ルートユーザーとIAMユーザー
		
		IAMユーザーで権限が無いもの（請求関連、アカウント削除など）は、一時的に
		ルートユーザーでログインして利用する。

		IAM
		ユーザー
		ユーザーを追加
		●ユーザー名
		各自、任意の名前を入力してください。
		《AWS マネジメントコンソールへのユーザーアクセスを提供する》にチェック。
		《IAMユーザーを作成します》にチェック。

		●コンソールパスワード
		ユーザ作成完了後にID・パスワード情報をダウンロードできます。
		《自動生成されたパスワード》にチェック。
		《ユーザーは次回のサインイン時に新しいパスワードを作成する必要があります(推奨)》にチェック。
		次へ
		
		許可を設定
		●許可のオプション
		《ユーザーをグループに追加》を選択。

		●グループの使用を開始する
		「グループを作成」をクリック。

		ユーザーグループを作成
		ユーザーグループを作成して、許可ポリシーを設定します。
		●ユーザーグループ名
		ここでは、”AdministratorGroup”とします。
		任意の名前を入力することができます。

		●許可ポリシー
		検索フォームに”AdministratorAccess”と入力して、”AdministratorAccess”(タイプ：AWS 管理 ? ジョブ機能)である項目にチェック。

		※「AdministratorGroup」に?入れてから次へ行かないと反映されなかった。

		「ユーザーグループを作成」をクリック。

		次へ

		パスワードを取得
		ここで「.csvファイルをダウンロード」をクリックして、認証情報を必ずダウンロードしてください。

		設定内容を確認
		IAMユーザーに紐付いているユーザーグループの許可ポリシーを確認します。
		IAM > ユーザーグループ > AdministratorGroup >「許可」タブ
		“AdministratorAccess”があることをご確認ください。

		IAMユーザーを作成することができました。

・	Rubyのバージョン管理をしよう

		Rubyバージョンを切り替えるソフトウェア
			RVM (Ruby Version Manager)　※コースで利用
			rbenv (Ruby Environment)
		
		バージョン確認
		rvm -v
		
		使えるRubyバージョン確認
		rvm list
		
		RVMインストール
		rvm get stable
		
		Rubyインストール
		rvm install ruby-3.0.2
		※最新をインストールする場合「rvm install ruby --latest」
		※なぜか最新がインストールされなかった3.0.0
		
		インストールされたRubyの確認
		rvm list
		
		次のコマンドを使用して、使用したいRubyのバージョンを指定します。
		rvm use <ruby_version>

		反映を確認
		ruby -v
		
		デフォルトで使用するバージョンを設定する
		rvm --default use 3.0.2

		表示が=*になっていることを確認
		rvm list
